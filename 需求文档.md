# Skill Manager - 需求文档 v2

> 参照 cc-switch 的技能管理功能，做一个**独立的技能安装/更新/卸载管理器**。
> 技术栈：Electron，打包成 Windows exe。

---

## 一、工具定位

一个跨工具的技能管理器。用户从 GitHub 仓库（或 skills.sh 注册表）发现技能，一键安装到目标 AI 工具的默认 skills 目录。

**与 cc-switch 的关系：** cc-switch 是个「全家桶」（配置切换 + 技能管理 + MCP + 会话等），这个工具**只做技能管理这一块**，做得更聚焦。

---

## 二、目标工具与安装路径

安装时用户自己选装到哪个工具，每个工具有固定的默认 skills 目录：

| 工具 | 默认 skills 目录 | 格式 |
|---|---|---|
| Claude Code | `~/.claude/skills/<技能名>/` | 原样复制 |
| OpenCode | `~/.config/opencode/skills/<技能名>/` | 原样复制 |
| OpenClaw | `~/.openclaw/skills/<技能名>/` | 原样复制 |
| Codex | `~/.codex/skills/<技能名>/` | 原样复制 |
| Hermes | `~/.hermes/skills/<技能名>/`（全局，与 cc-switch 一致） | 原样复制 |
| Trae CN | `<项目>/.trae/rules/<技能名>.md` | 格式转换（Cursor 风格单文件） |

> 前5个工具格式一致，直接复制技能文件夹即可。
> Trae CN 是例外：需要把 SKILL.md 转换成单个 .md 规则文件。

> **Hermes 说明：** 用户问"有全局的吗"——有。Hermes 除了 profile 级目录
> (`%LOCALAPPDATA%\hermes\profiles\<profile>\skills\`) 外，
> 还有全局目录 `~/.hermes/skills/`。默认装全局，与 cc-switch 一致。

## 三、功能模块

### 模块一：仓库管理

参照 cc-switch 的仓库管理逻辑：

- **添加仓库：** 填写 Owner（GitHub 用户名/组织）+ Name（仓库名）+ Branch（分支，默认 main）+ Subdirectory（技能所在子目录，可选）→ 添加到仓库列表
- **仓库列表：** 显示已添加的仓库（Owner/Name、分支、技能数量）
- **删除仓库：** 删除仓库（该仓库的技能不再出现在列表，已安装的不受影响）
- **skills.sh 搜索（可选进阶）：** 内置 skills.sh 公共注册表搜索，输入关键词实时搜索社区技能，点击直接添加

### 模块二：技能浏览

- 点击某个仓库 → 扫描仓库内技能（在配置的 Subdirectory 下递归扫描）
- **技能识别规则：** 一个文件夹内只要有 SKILL.md（或 README.md/prompt.md）就认为是一个技能
- **卡片展示：** 技能名、描述、来源仓库、安装状态
- **搜索框：** 按名称/描述/目录名实时搜索
- **状态过滤：** 全部 / 已安装 / 未安装
- **刷新：** 重新拉取仓库，获取最新技能列表

### 模块三：安装/卸载

- **安装：** 点击技能卡片上的「安装」→ 选目标工具（可多选）→ 安装到对应目录
- **安装方式（参照 cc-switch）：**
  - **软链接（Symlink，默认）：** 创建符号链接指向技能源文件，占用空间小，仓库 git pull 后更新实时同步到各工具
  - **复制（Copy）：** 将技能文件完整复制到目标目录
  - 在设置页面切换，默认推荐软链接
- **同名冲突：** 弹窗提示"技能已存在，是否覆盖？"，用户自己选择覆盖或取消
- **卸载：** 点击已安装技能的「卸载」→ 确认 → 从工具目录删除（软链接则删链接，复制则删文件夹）
- **卸载备份（参照 cc-switch）：** 删除前自动备份到本地缓存目录
- **安装状态标记：** 卡片上显示「已安装到 Claude Code」等标签

### 模块四：更新管理

参照 cc-switch v3.13.0 的更新机制：

- **更新检测：** 基于 SHA-256 内容哈希，对比本地已安装技能与远端仓库版本
- **单项更新：** 有新版的技能卡片显示「更新」按钮，点击即更新
- **全部更新：** 顶部「全部更新」按钮，批量更新所有有新版的技能

### 模块五：备份与恢复

参照 cc-switch 的备份恢复机制：

- **自动备份：** 卸载技能前，自动把技能文件夹备份到本地缓存目录（`~/.skill-manager/backups/`）
- **恢复：** 从备份列表选择，一键恢复（显示技能名 + 备份时间 + 来源工具）
- **删除备份：** 手动清理旧备份，释放空间
- **备份列表入口：** 设置页面或侧边栏入口

### 模块六：导入与导出

- **导出：** 把当前配置（仓库列表 + 已安装技能清单）导出成一个 JSON 文件
  - 包含：仓库配置（Owner/Name/Branch/Subdirectory）、已安装技能名称 + 安装到的工具列表
  - 用途：换机迁移、配置分享
- **导入：** 导入 JSON 文件，自动还原
  - 还原仓库列表
  - 提示哪些技能需要重新安装（因为技能文件不在 JSON 里，需要重新从仓库拉取）
- **入口：** 设置页面

### 模块七：日志管理

参照 cc-switch 的日志配置：

- **启用/关闭：** 可开关日志记录
- **日志级别（5级）：** error / warn / info / debug / trace
  - error — 仅记录错误
  - warn — 警告和错误
  - info — 一般信息（推荐，默认）
  - debug — 调试信息
  - trace — 所有详细信息
- **日志内容：** 安装/卸载/更新/导入导出/仓库添加删除等操作 + 时间 + 结果 + 错误堆栈（如有）
- **入口：** 设置页面 → 日志配置
- **日志存储位置：** `~/.skill-manager/logs/`

### 模块八：配置文件目录

参照 cc-switch 的目录配置 + 高级设置：

**工具自身目录（可自定义）：**

| 目录 | 默认路径 | 说明 |
|---|---|---|
| 应用数据目录 | `~/.skill-manager/` | 工具所有数据的根目录 |
| 仓库缓存 | `~/.skill-manager/repos/` | clone 下来的仓库 |
| 备份目录 | `~/.skill-manager/backups/` | 卸载时自动备份 |
| 日志目录 | `~/.skill-manager/logs/` | 操作日志 |

**各 AI 工具的配置目录（可自定义）：**

| 工具 | 默认路径 |
|---|---|
| Claude Code | `~/.claude/` |
| OpenCode | `~/.config/opencode/` |
| OpenClaw | `~/.openclaw/` |
| Codex | `~/.codex/` |
| Hermes | `~/.hermes/` |

> 与 cc-switch 一致：支持自定义各工具的配置目录路径（不写死，用户可改）。
> 修改目录后提示重启生效。

**目录操作：**
- 每个目录旁有「打开」按钮，一键用文件管理器打开
- 工具自身目录显示占用空间

## 五、技术方案

| 项 | 选择 |
|---|---|
| 框架 | Electron |
| 前端 | 原生 HTML/CSS/JS（不引框架，保持轻量） |
| 构建/打包 | electron-builder → Windows exe |
| Git 操作 | simple-git（Node.js 库） |
| 哈希计算 | Node.js crypto（SHA-256） |
| 本地配置存储 | electron-store |

### 目录结构

```
skill-manager/
├── package.json
├── src/
│   ├── main/              # Electron 主进程
│   │   ├── index.js        # 入口，创建窗口
│   │   ├── ipc.js          # IPC 处理（仓库/技能/安装）
│   │   ├── git.js          # Git 操作封装（clone/pull）
│   │   ├── skills.js       # 技能扫描/解析/安装/卸载/更新
│   │   └── store.js        # 本地配置存储（仓库列表等）
│   ├── preload/
│   │   └── index.js        # 安全桥接，暴露 API 给前端
│   └── renderer/           # 前端界面
│       ├── index.html
│       ├── styles.css
│       └── app.js
```

## 六、需求确认清单

| 问题 | 结论 |
|---|---|
| Hermes 有全局 skills 吗？ | ✅ 有，`~/.hermes/skills/`，默认装全局 |
| Trae CN 装到哪？ | 项目级 `.trae/rules/`，安装时用户选项目目录 |
| 仓库技能结构？ | 不强制结构，递归扫描含 SKILL.md 的文件夹（与 cc-switch 一致） |
| 同名技能冲突？ | 弹窗提示，用户选覆盖或取消 |
| 更新机制？ | SHA-256 内容哈希对比 |
| 卸载备份？ | 自动备份到 `~/.skill-manager/backups/` |
| 备份恢复？ | ✅ 从备份列表选择恢复 |
| 导入导出？ | ✅ 仓库列表 + 已安装清单导出为 JSON，导入还原 |
| 日志管理？ | ✅ 5级日志（error/warn/info/debug/trace），可开关，与 cc-switch 一致 |
| 配置目录？ | ✅ 工具自身 + 各 AI 工具目录均可自定义，一键打开，与 cc-switch 一致 |

## 七、cc-switch 功能对照（供参考）

调研 cc-switch 设置文档后，以下是 cc-switch 有但我们**暂不纳入**的功能（如果需要可以加）：

| cc-switch 功能 | 说明 | 是否需要？ |
|---|---|---|
| 主题设置（深色/浅色/跟随系统） | 界面主题切换 | ❓ 暂定只有深色 |
| 语言设置（中/英/日） | 多语言界面 | ❓ 暂定只有中文 |
| 开机自启 | 系统启动自动运行 | ❓ |
| 软链接 vs 复制安装 | cc-switch 默认用软链接，省空间、实时同步 | ❓ 暂定复制 |
| 关闭行为（最小化到托盘/直接退出） | 窗口关闭行为 | ❓ |
| 检测 CLI 工具版本 | 自动检测各 CLI 工具是否安装、版本号 | ❓ |
